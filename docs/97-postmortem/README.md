## 📂 事件结构

每个事件目录包含：
- `summary.md` - 事件概要
- `pm.md` - PM 视角的分析
- `arch.md` - 架构师的根本原因分析
- `test.md` - 测试经理的预防措施

## 📝 Postmortem 编写原则

1. **无责备文化** - 关注系统而非个人
2. **5 个为什么** - 深挖根本原因
3. **可执行的行动项** - 具体的改进措施
4. **时间线** - 事件发生、发现、解决的时间线
