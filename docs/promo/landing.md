## 三个卖点

### 🧩 可视化编排，不写后端
拖节点、连线、配 JSONPath 规则，就完成了一套"消息处理后端"。告别搭服务、接库、写路由。

### ⚙️ 生产能力开箱即用
幂等（同一业务键不重复处理）、应用级重试、运行记录可下钻、配置历史可回滚——平台都替你做了。

### 🔌 产物是你自己的 API，可复用
发布即可调用；做好的 API 能被其他工作流当积木直接调。还有 AI 帮你**生成**和**解释**工作流。

## 怎么用（三步）
1. **建应用**——它就是你的 API。
2. **画工作流**——节点是步骤，连线是顺序，JSONPath 是条件。
3. **发布并调用**——`POST` 一下，结果在运行记录里可查。

## FAQ
**要写代码吗？** 不用。画布编排 + JSONPath 规则即可；需要时可导入/导出工作流 JSON。

**做好的 API 怎么被复用？** 它就是一个 HTTP API；在别的工作流里用一个 CONSUMER 节点指向它即可组合。

**调用结果怎么拿？** 当前在**运行记录**里查看（按 `confirmationNumber` 筛选）；"由工作流塑造同步响应"在路线图上。

**支持哪些步骤？** HTTP 丰富化、条件分支(IFELSE)、消息下发、函数步骤等 6 种节点类型。
