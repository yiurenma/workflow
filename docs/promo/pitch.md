# 定位与电梯演讲

## 一句话
**Workflow Studio —— Serverless Easy API Maker。** 不写后端代码，在画布上把"消息丰富化 + 条件判断 + 多通道下发"编排成一个可调用的 API。

## 电梯演讲（30 秒）
> 做集成时，常要写一类后端：收到消息→调几个接口补全数据→按条件分支→下发到短信/邮件，还要做幂等、重试、留记录。这套东西每次都重写一遍，又慢又重。
>
> Workflow Studio 把它变成"画布编排 + 一键发布"：节点是步骤，连线是顺序，JSONPath 是条件，平台负责执行、幂等、重试和可追溯记录。发布之后，你的工作流就是一个 API——而且任何人都能调它、复用它。

## 目标人群
- **集成工程师**：要快速搭"消息处理后端"，不想每次造轮子。
- **应用开发者**：要一个稳定、可幂等调用的内部 API。
- **平台/中台团队**：要把一批集成逻辑标准化、可视化、可追溯。

## 价值主张（卖点三连）
1. **不写后端**：可视化编排替代搭服务/接库/写路由。
2. **内置生产能力**：幂等、重试、运行记录、配置历史/回滚开箱即用。
3. **产物是你自己的 API**：发布即可调用，且能被其他工作流当积木复用；还有 AI 帮你生成和解释。

## 对比一句话
| 对手 | 我们的差异 |
|---|---|
| 手写后端 | 你省掉搭服务/幂等/重试/运维，分钟级出 API |
| Lambda + API Gateway | 逻辑可视化、无需写代码配网关，内置记录与幂等 |
| Zapier / n8n | 聚焦"消息丰富化 + 条件下发 + 可追溯记录"，产物是你自己的 API |

## 诚实的边界（避免反噬）
当前调用结果在**运行记录**里查看（响应体为空）、统一入口、暂无内置入站鉴权——这些在路线图上。定位用户：**重在"编排并发布消息处理 API"**，而非"任意同步计算服务"。

## 一段话版本（可直接用于介绍）
Workflow Studio 是一个 Serverless Easy API Maker：在可视化画布上用节点和 JSONPath 规则编排"丰富化→条件→下发"，一键发布成带幂等、重试和运行记录的 API，无需写后端；做好的 API 可被其他工作流直接复用，并有 AI 辅助生成与解释。
