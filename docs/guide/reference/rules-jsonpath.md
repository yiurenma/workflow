# 规则语言 — JSONPath

规则是节点的**守卫**：对当前[运行负载](../03-concepts.md)求值，**命中才执行该步**，否则跳过（无隐式默认分支）。

## 规则的形态
每个节点挂一个 `ruleList`，每条规则：
```json
{ "key": "$.customerId", "remark": "有 customerId 才执行" }
```
- `key`：**单个有效的 JSONPath 表达式**（用 `jsonpath-plus` 校验）。
- `remark`：人类可读说明。
- 一个节点的多条规则需**全部匹配**才执行该步。

## 硬性约束
- `key` 必须是**单个** JSONPath：**不能**含逗号 / 分号 / 多个路径，**不能**是无 JSONPath 语法的自由文本。
- 失焦时校验；语法错误时**保存按钮禁用**，并提示如：
  > 规则键必须是单个有效的 JSONPath 表达式（例如 `$.data.amount`）

## 语义
- "匹配" = 该 JSONPath 在运行负载上**求值到非空结果**。
- 全部规则匹配 → 执行该步；任一不匹配 → 跳过该步。
- IFELSE：命中走 `true` 分支，否则按 `elseLogic` 回退。

## 常用写法
| 目的 | JSONPath |
|---|---|
| 字段存在 | `$.customerId` |
| 嵌套字段 | `$.profile.contact` |
| 数组首元素 | `$.items[0]` |
| 按条件过滤 | `$.items[?(@.amount > 1000)]` |
| 深层任意位置 | `$..email` |

## 常见错误
| 写法 | 问题 |
|---|---|
| `$.a, $.b` | ❌ 多路径（含逗号） |
| `amount > 1000` | ❌ 自由文本，非 JSONPath |
| `$.a; $.b` | ❌ 含分号 |
| `$.data.amount` | ✅ 单个有效路径 |

## 提示
- 规则判断"**要不要做这步**"，不做数据转换/赋值。需要算术/字符串/映射时，目前放在 HTTP 请求体模板或函数步骤里完成（通用表达式语言在路线图上）。
- 占位符/帮助文本会给出有效 JSONPath 示例。
